Algorithms

To make a computer do anything, you have to write a computer
program. To write a computer program, you have to tell
the computer, step by step, exactly what you want it to do.
The computer then "executes" the program, following each step mechanically, to accomplish the end goal.
When you are telling the computer what to do, you also get to choose how it's going to do it.
That's where computer algorithms come in. The algorithm is the basic technique used to get the job done.
Let's follow an example to help get an understanding of the algorithm concept.

An algorithm is a defined computational procedure that takes some values, or a set of values, often known as
input and produces some value, or a set of values as output.

Example: One might need to sort a sequence of numbers into non-decreasing order.
- This problem is highly demand and is done very often, and because of that there exists know and pre-coded algorithms
to do excatly what you need. The recepice is already there, therefore is no need to figure out a new algorithm for something
that already exists.
Various algorithms might be Merge Sort, Quick Sort, Heap Sort, Bin Sort, etc.
What is needed on the other hand, is to know each algorithm to be able to know exactly when to use which.

For example, if you have a million integer values between 1 and 10 and you need to sort them, the Bin Sort
is the correct algorithm to use. If you have a million books titles, then quick sort might be a better option.
The important aspect here is to know the strenghts and weaknesses of the different algorithms!

Know you might be wondering.. If that is algorithm, then what is data structures..?
If you are, that is very nice, because I am about to tell you the difference between the two right now.

"DIFFERENCE BETWEEN DATA STRUCTURE AND AN ALGORITHM: A data structure is the way you define a certain object
in a program language. ... An algorithm is a specific way of programming a task to make it work on the data you have".
